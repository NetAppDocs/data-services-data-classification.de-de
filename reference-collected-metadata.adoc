= Aus Datenquellen in der NetApp Data Classification erfasste Metadaten
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
NetApp Data Classification erfasst Metadaten beim Scannen Ihrer Datenquellen und Systeme. Die meisten Metadaten sind verfügbar, aber einige Attribute sind nicht für alle Protokolle verfügbar.

[cols="15,25,25,25"]
|===
|  | *Metadaten* | *CIFS* | *NFS* 


.3+| *Zeitstempel* | _Erstellungszeit_ | Verfügbar | Nicht verfügbar (wird unter Linux nicht unterstützt) 


| _Letzter Zugriffszeitpunkt_ | Verfügbar | Verfügbar 


| _Letzte Änderungszeit_ | Verfügbar | Verfügbar 


.2+| *Berechtigungen* | _Öffnen Sie Berechtigungen_ | Wenn die Gruppe „JEDER“ Zugriff auf die Datei hat, gilt sie als „Für die Organisation offen“. | Wenn „Andere“ Zugriff auf die Datei haben, gilt sie als „Für die Organisation offen“. 


| _Benutzer-/Gruppenzugriff_ | Benutzer- und Gruppeninformationen werden aus LDAP übernommen | Nicht verfügbar (NFS-Benutzer werden normalerweise lokal auf dem Server verwaltet, daher kann dieselbe Person auf jedem Server eine andere UID haben) 
|===
[NOTE]
====
* Die Datenklassifizierung extrahiert nicht die „Zeit des letzten Zugriffs“ aus den Datenbankdatenquellen.
* Ältere Versionen des Windows-Betriebssystems (zum Beispiel Windows 7 und Windows 8) deaktivieren die Erfassung des Attributs „letzter Zugriffszeitpunkt“ standardmäßig, da dies die Systemleistung beeinträchtigen kann. Wenn dieses Attribut nicht erfasst wird, werden Data Classification Analysen, die auf dem „letzten Zugriffszeitpunkt“ basieren, beeinträchtigt. Die Erfassung des letzten Zugriffszeitpunkts kann auf diesen älteren Windows-Systemen bei Bedarf aktiviert werden.


====


== Letzter Zugriffszeitpunkt

Wenn NetApp Data Classification Daten aus Dateifreigaben extrahiert, wertet das Betriebssystem dies als Zugriffsereignis und aktualisiert die „letzte Zugriffszeit“ entsprechend. Nach dem Scan versucht NetApp Data Classification, die letzte Zugriffszeit auf den ursprünglichen Zeitstempel zurückzusetzen. Wenn NetApp Data Classification keine Schreibattributberechtigungen in CIFS oder Schreibberechtigungen in NFS hat, kann das System die letzte Zugriffszeit nicht auf den ursprünglichen Zeitstempel zurücksetzen.

Standardmäßig scannt NetApp Data Classification Dateien nicht, wenn keine Schreibberechtigung zum Wiederherstellen des ursprünglichen Zeitstempels vorhanden ist. Falls Ihre Daten es tolerieren, dass die zuletzt aufgerufene Zeit nicht auf den ursprünglichen Zeitstempel zurückgesetzt wird, kann die Option *Scan without write permissions* auf der Systemkonfigurationsseite aktiviert werden.

image:screenshot_scan_missing_permissions.png["Ein Screenshot, der den Schalter zeigt, mit dem Data Classification Ihre Dateien scannen kann, selbst wenn nicht die erforderlichen Berechtigungen vorhanden sind, um die zuletzt aufgerufene Zeit auf den ursprünglichen Zeitstempel zurückzusetzen."]

Diese Funktionalität ist auf lokale ONTAP Systeme, Cloud Volumes ONTAP, Azure NetApp Files, Amazon FSx for NetApp ONTAP Management und Dateifreigaben von Drittanbietern anwendbar.


NOTE: ONTAP Volumes, die mit SnapLock konfiguriert sind, verfügen über reine Leseberechtigungen und können den Zeitpunkt des letzten Zugriffs auch nicht auf den ursprünglichen Zeitstempel zurücksetzen.



=== Scan-Analyseereignisse

Auf der Seite „Untersuchung“ gibt es einen Filter namens „Scan Analysis Event“, mit dem entweder die Dateien angezeigt werden, die nicht klassifiziert wurden, weil NetApp Data Classification den letzten Zugriffszeitpunkt nicht zurücksetzen konnte, oder die Dateien, die klassifiziert wurden, obwohl NetApp Data Classification den letzten Zugriffszeitpunkt nicht zurücksetzen konnte.

image:screenshot_scan_analysis_event_filter.png["Ein Screenshot des Filters, um zu sehen, welche Dateien gescannt und welche nicht gescannt wurden, je nachdem, ob sie Schreibberechtigungen hatten oder nicht."]

Die Filterauswahl ist:

* „Nicht klassifiziert -- Letzter Zugriffszeitpunkt kann nicht wiederhergestellt werden“ - Hier werden die Dateien angezeigt, die aufgrund fehlender Schreibberechtigungen nicht klassifiziert wurden.
* „Klassifiziert und zuletzt aufgerufen“ – Hier werden die Dateien angezeigt, die klassifiziert wurden und deren zuletzt aufgerufener Zeitpunkt von NetApp Data Classification nicht auf das ursprüngliche Datum zurückgesetzt werden konnte. Dieser Filter ist nur für Umgebungen relevant, in denen *Scannen ohne Schreibberechtigung* aktiviert ist.


Bei Bedarf ist ein Export dieser Ergebnisse in einen Bericht möglich, sodass ersichtlich ist, welche Dateien aufgrund von Berechtigungen gescannt werden oder nicht. link:task-investigate-data.html#download-your-report["Erfahren Sie mehr über Data Investigation-Berichte"^].
